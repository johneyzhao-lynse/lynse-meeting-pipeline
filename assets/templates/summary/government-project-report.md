# 模板说明

适用于政府项目汇报、专项工作汇报、政企合作汇报、阶段验收汇报和领导沟通会，用来整理项目背景、建设进展、阶段成果、问题风险和下一步落实安排。

# 总结目标

帮助项目团队形成清晰、稳妥、可汇报的工作总结，明确项目成果、待协调事项、责任分工和后续推进重点。

# 目标读者

政府部门、项目建设单位、承建单位、实施团队、管理者和需要参与汇报或落实的人。

# 输出结构

## 项目概况

整理项目背景、主管单位、建设目标、服务对象和当前阶段。

## 工作进展

按模块、阶段或任务线整理已完成工作、正在推进事项和阶段性成果。

## 关键成果

提炼可汇报的成果、数据、案例、机制建设和社会或业务价值。

## 问题与风险

整理政策、审批、资金、进度、协同、数据、安全或验收方面的问题。

## 协调事项

列出需要上级部门、合作单位、供应商或内部团队协调解决的事项。

## 下一步安排

整理后续工作计划、责任单位、时间节点、交付物和验收准备。

# 特殊要求

- 表述稳妥、客观，避免夸大成果或承诺未确定事项。
- 对主管单位、时间节点、关键成果、风险和责任单位加粗。
- 不编造政策依据、项目数据、验收结论或领导意见。
- 区分已完成、进行中、待协调和待决策事项。
